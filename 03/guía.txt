¿Qué son las Pruebas Unitarias?
Una prueba unitaria (unit test) es un procedimiento automatizado que verifica el comportamiento de la unidad
más pequeña de código comprobable en una aplicación, generalmente una función o método, de forma aislada
del resto del sistema. Las pruebas unitarias constituyen la base de la pirámide de pruebas (Cohn, 2010) y son el
mecanismo más rápido y económico para detectar regresiones durante el desarrollo.
Según Osherove (2013), una buena prueba unitaria posee tres atributos fundamentales: (1) es completamente
automatizada, (2) su resultado es determinista (siempre produce el mismo resultado para los mismos datos), y
(3) prueba una sola unidad lógica.

¿Qué es Pytest?
Pytest es el framework de pruebas más popular del ecosistema Python (Okken, 2022). A diferencia del módulo
estándar unittest, Pytest no requiere que las pruebas hereden de una clase base; basta con escribir funciones
cuyo nombre comience con test_. Pytest detecta automáticamente estos archivos y funciones mediante su
sistema de descubrimiento (discovery).
Sus ventajas principales son:
●
●
●

Sintaxis minimalista: se usan assertions nativas de Python (assert) en lugar de métodos especiales.
Mensajes de error detallados: al fallar un assert, Pytest muestra el valor real vs. el esperado.
Pruebas parametrizadas: el decorador @pytest.mark.parametrize permite ejecutar la misma lógica con
múltiples conjuntos de datos.
Fixtures: mecanismo para preparar y limpiar contexto de prueba de forma reutilizable.
Plugins: ecosistema extensible con más de 1 000 plugins (pytest-cov, pytest-html, etc.).

●
●

Instalación:
pip install pytest
pytest --version # Verificar instalación

El Patrón AAA: Arrange – Act – Assert
El patrón AAA (Beck, 2002) es la estructura canónica de cualquier prueba unitaria:

Fase

Descripción

Arrange

Prepara los datos de entrada, objetos y condiciones previas necesarias para la
prueba.

Act

Ejecuta la unidad bajo prueba (función o método) con los datos preparados.

Assert

Verifica que el resultado obtenido coincide con el resultado esperado.

Ejemplo mínimo ilustrativo:
# test_suma.py
def sumar(a, b):
return a + b

Pruebas Parametrizadas con @pytest.mark.parametrize
Cuando una función debe probarse con múltiples combinaciones de entrada/salida, el decorador
@pytest.mark.parametrize evita la duplicación de código. Esto es especialmente relevante cuando se aplican
técnicas de diseño de casos de prueba como la Partición de Equivalencia o el Análisis de Valores Límite (Spillner
& Linz, 2021).
import pytest

@pytest.mark.parametrize("a, b, esperado", [
(2,

3,

5),

# caso normal

(0,

0,

0),

# caso borde: ceros

(-1, 1,

0),

# caso borde: negativos

(100, 200, 300),# caso grande
])
def test_sumar_parametrizado(a, b, esperado):
assert sumar(a, b) == esperado

Fixtures en Pytest
Un fixture es una función decorada con @pytest.fixture que proporciona un contexto de prueba reutilizable. Se
utiliza cuando varias pruebas comparten la misma configuración inicial (p.ej., crear un objeto, conectar a una
base de datos, preparar datos de ejemplo). Pytest inyecta el fixture automáticamente al declararlo como
parámetro de la función de prueba (Okken, 2022).

Convenciones de nomenclatura en Pytest
Para que Pytest descubra automáticamente los archivos y funciones de prueba, se deben seguir estas
convenciones:
●
●
●

Archivos de prueba: deben nombrarse test_*.py o *_test.py
Funciones de prueba: deben comenzar con test_
Clases de prueba (opcional): deben comenzar con Test (sin método __init__)

def celsius_a_fahrenheit(celsius):
"""Convierte grados Celsius a Fahrenheit.
Fórmula: F = (C * 9/5) + 32
"""
return (celsius * 9 / 5) + 32

def fahrenheit_a_celsius(fahrenheit):
"""Convierte grados Fahrenheit a Celsius.
Fórmula: C = (F - 32) * 5/9
"""
return (fahrenheit - 32) * 5 / 9
